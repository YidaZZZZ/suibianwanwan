# 对话模板 - 新对话快速启动

## 📝 如何创建新对话

### 方法1: 使用脚本（推荐）

```bash
node e:/AIworkbuddy/workDir/agent-config/scripts/create-conversation.js --date 2026-04-01 --topic "topic-name"
```

### 方法2: 手动创建

1. 在 `conversations/` 下创建日期目录：`YYYY-MM-DD/`
2. 复制此模板到新目录
3. 更新 `notes.md` 记录对话内容

---

## 📁 对话标准结构

```
conversations/YYYY-MM-DD/
├── notes.md              # 对话笔记（必需）
├── experiments/           # 实验代码
│   ├── experiment-1.js   # 实验1
│   ├── experiment-2.js   # 实验2
│   └── ...
├── tests/                # 测试文件
│   ├── test-1.js
│   └── ...
├── assets/               # 资源文件
│   ├── images/
│   └── screenshots/
├── snippets/             # 代码片段
│   ├── snippet-1.js
│   └── ...
└── archive.md            # 归档说明
```

## 📝 notes.md 模板

```markdown
# 对话记录 - YYYY-MM-DD

## 📋 主题

[简述对话主题]

---

## 💬 主要内容

### 1. [主题1]
- 关键点1
- 关键点2

### 2. [主题2]
- 关键点1
- 关键点2

---

## 🗂️ 文件记录

### 创建的文件
- `file1.js` - 说明
- `file2.md` - 说明

### 修改的文件
- `existing-file.js` - 修改内容

### 删除的文件
- `old-file.js` - 删除原因

---

## 💡 关键洞察

[记录重要的发现和洞察]

---

## 📊 统计信息

- **对话轮次**: N
- **创建文件数**: N
- **代码行数**: N

---

**记录时间**: YYYY-MM-DD
```

## 🎯 对话类型

### 1. Development（开发对话）
- 代码实现
- Bug修复
- 功能开发

### 2. Experiment（实验对话）
- 新想法验证
- 技术探索
- 概念验证

### 3. Design（设计对话）
- 架构设计
- 系统规划
- 方案讨论

### 4. Debug（调试对话）
- 问题排查
- 错误分析
- 性能优化

---

## 📋 归档流程

### 1. 对话进行中
- 实时代码放在 `experiments/`
- 重要发现记在 `notes.md`
- 临时资源放 `assets/`

### 2. 对话结束
- 整理 `notes.md`
- 归档有价值的代码到 `projects/`
- 删除临时文件
- 编写 `archive.md` 总结

### 3. 定期清理
- 每周整理实验代码
- 每月清理过期对话
- 重要代码迁移到项目

---

## 🔄 内容流转图

```
对话产生代码
    ↓
是否有价值？
    ├─ 是 → 是否完整项目？
    │       ├─ 是 → 移动到 projects/
    │       └─ 否 → 保留在 conversations/
    └─ 否 → 删除
```
