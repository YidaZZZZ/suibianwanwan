# Agent 载入协议 v2

> 使用方式：在对话中说 **"载入 XXX"**（XXX 是 agent 的 md 文件名，不含 .md 后缀）
> 例如："载入 Dao"、"载入 能力扩展优先Agent"

---

## 设计哲学

**全局 = 基础模板**（通用原则、通用能力框架）
**项目级 = 具体化**（针对当前工作区的特化、补充、覆盖）

类比：全局是"人"的定义，项目级是"这个人在这份工作中的具体角色"。

---

## 载入流程

当用户说"载入 XXX"时，按以下顺序执行：

### 第 1 步：读取全局 Agent 配置（基础模板）

先检查项目级 Agent 的 `extends` 字段，确定基础模板名称。

```
基础模板路径: c:\Users\guess\.codebuddy\agents\{extends的值}.md
```

- 如果项目级 Agent 有 `extends` 字段 → 读取其指定的全局模板作为基础层
- 如果没有 `extends` → 尝试读取同名的全局模板 `c:\Users\guess\.codebuddy\agents\XXX.md`
- 都不存在 → 跳过，基础层为空

### 第 2 步：读取项目级 Agent 配置（具体化层）

```
路径: {当前工作目录}\.codebuddy\agents\XXX.md
```

- 存在 → 读取，作为**具体化层**
- 不存在 → 只有基础层生效（纯通用模式）

### 第 3 步：合并规则（基础 + 具体化）

```
最终配置 = 基础层 + 具体化层（补充新增 / 覆盖冲突）
```

| 层级 | 角色 | 作用 |
|------|------|------|
| 基础层（全局） | 通用原则 | 提供核心身份、通用行为准则、通用元认知 |
| 具体化层（项目级） | 特化补充 | 添加项目专属原则、覆盖通用原则、定义项目专属行为 |

**合并策略**：
- `核心身份` → 基础层提供，具体化层可覆盖
- `核心原则` → 基础层 + 具体化层合并，具体化层新增的追加
- `行为准则` → 基础层 + 具体化层合并
- `元认知问题` → 合并去重，具体化层的问题优先
- `执行优先级` → 具体化层覆盖基础层
- `frontmatter` → 具体化层覆盖基础层

### 第 4 步：按需读取项目配置数据

合并后的 Agent 配置可能引用 `agent-config/configs/` 下的数据文件。
根据 Agent 内容中的关键词**按需读取**：

| 触发关键词 | 读取文件 |
|------------|---------|
| 技能/能力/能力扩展 | capabilities-log.json |
| 核心驱动/资源获取 | core-drives.json |
| 思考/元认知/思考模式 | thinking.json |
| 记忆/技能转化 | memory.json, skills-ideas-management.json |
| 模型/token成本 | ai-models.json |

**原则**：只读必要的文件，控制 token 消耗。

### 第 5 步：确认载入

回复格式：
```
✅ 载入完成: XXX

📋 配置层级:
- 基础层(全局): c:\Users\guess\.codebuddy\agents\XXX.md ✅/❌
- 具体化层(项目): .codebuddy\agents\XXX.md ✅/❌

🧠 核心身份: [合并后的一句话总结]
🎯 核心原则: [列出合并后的原则]
📊 技能库: X 个技能, Y 个能力里程碑
🔍 当前焦点: [focus_directions]

准备就绪。
```

## 快捷指令

| 指令 | 效果 |
|------|------|
| `载入 XXX` | 载入指定 Agent（全局基础 + 项目级具体化） |
| `载入列表` | 显示所有可用 Agent |
| `载入状态` | 显示当前已载入的 Agent 和配置 |
| `写入技能` | 将本次对话获得的新技能追加到 capabilities-log.json |

## 创建新 Agent 的规范

### 全局基础模板格式
```markdown
---
name: XXX
description: 一句话描述通用定位
model: glm-5.0-turbo
tools: [工具列表]
agentMode: manual
enabled: true
enabledAutoRun: true
---
# XXX - 基础模板

## 核心身份（通用）
[这个类型 Agent 的通用定义]

## 核心原则（通用）
[适用于所有项目的通用原则]

## 行为准则（通用）
[通用行为框架]

## 元认知（通用）
[通用自问问题]
```

### 项目级具体化格式
```markdown
---
name: XXX
description: 一句话描述项目级特化
model: [可覆盖]
tools: [可覆盖]
---
# XXX - [项目名]具体化

## 核心身份（特化）
[在本项目中的具体角色]

## 新增原则
[项目专属原则]

## 覆盖原则
[覆盖通用原则的项目专属版本]

## 项目专属行为
[针对本项目的具体行为规则]

## 元认知（补充）
[项目专属的自问问题]
```
